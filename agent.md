# MyClaw Agent 前端协作约定

本文件用于约束 Agent 在 Web 前端改动时的交互与发布方式。

## 资源版本号策略（强制）

- 每次修改前端静态资源时，必须同步更新引用版本号，避免浏览器缓存旧文件。
- 适用范围：HTML 中引用的 JS/CSS。
- 推荐方式：在 URL 追加查询参数版本号，如：
  - /static/app.js?v=20260513-8
  - /static/styles.css?v=20260513-5
- 若同次仅改 JS，至少更新 JS 版本；若改 CSS，至少更新 CSS 版本。

## 交互方式约定（强制）

- 禁止使用浏览器阻断式弹窗：
  - window.alert
  - window.confirm
  - window.prompt
- 需要用户确认或输入时，优先使用页面内交互组件，不打断主流程。

## 现代化 Web 交互建议（优先）

- 优先原地编辑（inline edit）替代弹窗输入。
- 输入提交流程建议：
  - Enter 提交
  - Escape 取消
  - blur 触发提交或根据场景回滚
- 异步提交时提供即时反馈：
  - 按钮禁用态/加载态
  - 成功或失败提示
- 避免一次点击触发重渲染导致后续双击/长按失效。
- 对关键交互元素增加可访问性属性（aria-label）与明确的 hover/focus 状态。

## 会话名称编辑规范（当前项目）

- 双击会话名称进入原地编辑。
- 单击会话名称不弹窗，不阻断页面。
- 编辑完成后调用后端接口更新名称，并刷新会话列表。
- 失败时在页面内给出非阻断式错误提示。
